State Management Quiz

Q1. Can post requests be bookmarked? What are the problems?

A1. An HTTP POST can be bookmarked, but since a bookmark only consists of the URL, all of the form parameters will be lost. This will often mean that the web server doesn't know what to do with the request, since it was expecting some form parameters.

Q2. What is the purpose of request dispatching?

A2. The purpose of request dispatching is dispatching requests to any resource (such as HTML, Image, JSP, Servlet etc.) on the server via Request Dispatcher interface.

Q3. What is the difference between redirect and request dispatch?

A3. I redirect; request is redirected to client (Browser) and it will process the new URL, end User can see on which page url is redirected and processing is done at client side. In dispatch; servlet will internally forward the request to another servlet or jsp page, end user don’t know that which page is processed internally and processing is done at server side.

Q4. What is an attribute ?

A4.An attribute is an object that is used to share information in a web app, it allows Servlets to share information among themselves.

Q5.What is the difference between attributes and parameters?

A5. Parameters may come into our application from the client request, or may be configured through deployment descriptor (web.xml) elements or their corresponding annotations. Attributes are objects that are attached to various scopes and can be modified, retrieved or removed. Parameters are read only, attributes are read/write objects. And parameters are String objects, attributes can be objects of any type.

Q6. What are dangers of using attributes?

A6. Context and session attributes are not thread safe.

Q7. What does it mean to say that http is stateless? Give an example of a stateful protocol.

A7. Http protocol is stateless in a sense that it does not require the server to retain information or status about each user for the duration of multiple requests. FTP is an example of stateful protocol and Http can be assumed stateful by using cookies.

Q8. Give 5 different methods for maintaining state information (count each attribute scope as one method).

A8. 5 different methods to maintain state are:

  1. request scope: destroyed when servlet finishes processing request
  2. session scope: destroyed when user closes browser
  3. application scope destroyed when the container stopped.
  4. Cookies saved on browser, either temporary (deleted when the browser closes) or permanent
  5. Hidden fields on a form

Q9. How long does a session last?

A9. A session can last for the period of session-timeout set by the DD(web.xml) or the setMaxInactiveInterval method set by the programmer. Also session can be invalidated manually by the programmer after completing some tasks.

Q10. What is a cookie, and how long does a cookie last?

A10. A cookie is a small piece of data stored on the client-side which servers use when communicating with clients. A cookie can last for a period of time configured by the programmer by a method setMaxAge().

Q11. What is the purpose of URL rewriting?

A11. Url rewriting is a process of appending or modifying any url structure while loading a page. If our browser does not support cookies, URL rewriting provides us with another session tracking alternative.

Q12. Why does the request attribute report ‘null’ for the maintaining state demo?

A12. Because as soon as the server has finished generating the response for the request, the request doesn't exist anymore.
